# 数据持久化存储

## 需求背景

TBOS 框架本身不绑定具体存储方案，厂商需自行实现数据持久化层，将采集到的设备数据、告警记录、配置信息等落盘存储。框架仅定义数据接口规范，具体存储选型（如 MySQL、TDengine、ClickHouse 等）由厂商自行决定。

## 阻塞原因

- TBOS 框架的数据接口规范尚未在开源版中完整发布
- 不同存储方案的接入适配代码需要框架侧配合
- 缺少典型的存储方案性能对比与推荐路径
