# 第三方系统对接

## 需求背景

运营商要求动环监控系统需与既有 IT 运维体系打通，厂商需完成与第三方系统（如 ITSM 工单系统、短信/邮件/企业微信告警通道、CMDB、北斗定位等）的对接集成。

## 阻塞原因

- 运营商未明确必须对接的第三方系统清单及版本
- 各第三方系统的 API 协议与认证方式差异大
- TBOS 开源版未提供统一的对接网关或适配器框架

## 待明确事项

- [ ] 必须对接的第三方系统清单与优先级排序
- [ ] 各系统 API 协议类型（REST / gRPC / WebSocket / MQTT 等）
- [ ] TBOS 是否提供统一告警通道抽象层
- [ ] 对接安全认证规范（Token / AK-SK / OAuth2 等）
