TBOS (T-Block Operation System) 是腾讯自研的机房动环监控系统，采用微服务架构设计，包含边缘采集、数据处理、告警计算、配置管理等多个核心模块，支持大规模机房的统一监控管理。

### 1. 核心能力

TBOS 围绕机房动环监控的完整链路构建，覆盖设备接入、数据采集、实时处理、告警计算、配置管理、数据查询与可视化等核心能力，适用于数据中心、边缘站点和工业设备的统一监控管理。

| 特性 | 说明 |
|------|------|
| 🔌 **多协议采集** | 支持 Modbus RTU/TCP、SNMP、DIO 等多种工业协议，可扩展驱动插件 |
| ⚡ **实时告警** | 毫秒级告警响应，支持实时告警(1秒)、延时告警(3秒)、虚拟测点告警(2秒) |
| 🎯 **TNQL表达式引擎** | 支持复杂逻辑、比较、算术运算，内置 DelayEQ/Rise/Fall/InRange 等函数 |
| 🔄 **分布式架构** | 支持水平扩展，高可用部署，主备 Kafka 双活自动切换 |
| 📊 **标准化测点** | 统一的测点体系，支持虚拟测点计算和表达式求值 |
| 📡 **WebSocket推送** | 实时告警推送，无需轮询，前端自动重连 |
| 🌐 **边云协同** | 边缘采集器与云端服务协同工作，支持配置本地缓存兜底 |
| ⚖️ **智能调度** | 基于计算复杂度的负载均衡任务调度，增量下发减少开销 |
| 🔥 **配置热更新** | 支持运行时动态更新采集配置和告警策略，无需重启服务 |


### 2. 产品优势

- **采集能力丰富**：支持 Modbus RTU/TCP、SNMP、DIO 等多种工业协议，能够对接不同厂家、不同型号的动环设备。通过采集模板和标准化测点体系，将协议测点统一映射为平台测点，降低设备接入和后续维护成本。

- **实时告警能力强**：提供秒级甚至毫秒级告警响应，内置 TNQL 表达式引擎，支持比较、逻辑、算术运算以及 DelayEQ、Rise、Fall、InRange 等函数，能够表达复杂的告警条件、恢复条件和虚拟测点计算逻辑。

- **边云协同架构**：边缘侧负责设备接入、协议采集和本地缓存，云端负责配置管理、任务调度、数据处理、告警计算和集中展示。在网络异常或配置服务不可用时，边缘侧可依赖本地缓存继续运行，提升系统可用性。

- **配置动态生效**：支持采集配置、计算任务和告警策略的热更新，Scheduler 监听配置变化并将任务增量下发到 Agent、Data-Compute 和 Alarm-Compute 等服务，避免频繁重启服务，降低运维风险。

### 4. 部署模式

TBOS 提供单体部署和集群部署两种模式。

**单体部署**适合开发调试、功能验证和集成测试。支持本地编译运行（详见[单机部署指南](./quick_start/standalone.md)）和 Docker 容器化两种方式。所有服务部署在单台主机上，不具备高可用能力，不建议用于生产环境。

**集群模式**面向生产环境与大规模机房接入场景，各服务可水平扩展为多实例分布式部署，外部组件（MySQL、Redis、Kafka、InfluxDB）均采用集群版本。支持边云协同、主备 Kafka 双活自动切换，可承载形态包括自建服务器、公有云和私有云（K8S 等）。
