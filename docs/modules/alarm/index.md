# 告警模块

告警模块是TBOS系统的核心监控组件，用于实施监控机房设备状态，分析测点数据，生成告警并进行全生命周期管理。主要包括以下功能：

- **定义[标准设备](../../concepts/device.md)**: 维护业务系统关注的标准设备类型字典
- **设备实例化**：维护一个业务设备表，为每设备分配唯一标识，并关联标准设备
- **采集配置**：维护采集服务对接采集接口（TBOX/厂商采集器）所需的配置，包含采集目标，采集测点等信息
- **[告警策略](./alarm_strategy.md)配置**：配置基于标准设备实例化的告警规则

告警模块采用流式架构，通过Kafka解耦，支持海量测点的高可靠性处理。

## 模块概览

告警模块包含以下 3 个核心服务：

| 服务 | 文档 | 说明 |
|------|------|------|
| **Alarm Compute** | [alarm-compute.md](alarm-compute.md) | 告警计算引擎，接收 Scheduler 下发的告警策略任务，实时计算测点数据判断是否触发告警 |
| **Alarm Manage** | [alarm-manage.md](alarm-manage.md) | 告警管理服务，接收告警消息并负责多通道推送（企业微信机器人等） |
| **Alarm Server** | [alarm-server.md](alarm-server.md) | 告警查询与操作服务，提供告警列表/详情/统计/状态管理等查询与操作接口 |

## 服务间关系

- **Alarm Compute** 接收 Scheduler 下发的告警策略，实时消费测点数据，根据策略表达式判断是否触发告警
- **Alarm Manage** 接收 Alarm Compute 触发的告警消息，通过企业微信机器人等渠道推送告警通知
- **Alarm Server** 提供告警数据的查询和操作能力（告警列表、详情、统计、状态更新、回放诊断等）
- 前端通过 **CGI 网关**（属于平台模块）统一访问告警接口
